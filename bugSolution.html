The solution addresses the MIME type issue within the <object> tag. Ensuring the correct MIME type application/pdf is specified, the browser correctly handles the PDF embedding.

<object data="data.pdf" type="application/pdf">  <p>Your browser does not support embedded PDFs.</p> </object>

Note: This solution assumes the data.pdf file exists and is accessible. In a real-world scenario, robust error handling and alternative content would be important for scenarios where the PDF is unavailable.
